# IT Project Manager — CRM Implementation

## Project Overview

This project simulates a real-world CRM implementation initiative led from the perspective of an IT Project Manager. The project focuses on planning, coordinating, and managing the rollout of a customer relationship management system to improve sales visibility, customer tracking, stakeholder communication, and operational reporting.

The project demonstrates how an IT Project Manager would define scope, manage implementation risks, align business and technical stakeholders, track milestones, monitor adoption, and support a smooth transition from current-state processes to a new CRM workflow.

## Business Problem

The organization was managing customer information, sales activities, follow-ups, and reporting across disconnected tools and manual spreadsheets. This created inconsistent customer data, missed follow-ups, limited sales pipeline visibility, duplicate work, and difficulty tracking team performance.

Leadership needed a structured CRM implementation plan that could improve data accuracy, centralize customer information, support sales and service teams, and create better reporting for managers.

## Project Goals

- Implement a CRM system to centralize customer and sales activity data
- Improve visibility into sales pipeline, follow-ups, and customer interactions
- Reduce manual tracking through standardized CRM workflows
- Increase user adoption through training, documentation, and stakeholder support
- Identify risks, dependencies, and change management needs before launch
- Create KPI reporting for leadership decision-making

## Key Deliverables

| Deliverable | Purpose |
|---|---|
| Executive Summary | Summarizes the implementation need, project approach, and expected business value |
| Project Charter | Defines project scope, objectives, timeline, assumptions, and success criteria |
| Stakeholder Map | Identifies business, technical, leadership, and end-user stakeholders |
| KPI Dashboard | Tracks implementation progress, adoption, data quality, and business performance |
| Risk Register | Documents implementation risks, impact, likelihood, owners, and mitigation plans |
| Action Plan | Breaks the CRM rollout into workstreams, owners, due dates, and completion status |
| Source Data | Supports KPI tracking for adoption, data quality, training, and implementation progress |
| Meeting Notes | Shows project communication, decisions, blockers, and follow-up actions |
| Implementation Playbook | Explains rollout phases, governance, training, support, and post-launch monitoring |
| Dashboard Readout | Interprets key implementation metrics and project health indicators |
| Resume & Interview Talking Points | Converts the project into job-ready language for interviews and resumes |

## Business Impact

This project demonstrates how an IT Project Manager can move a business from manual, disconnected customer tracking into a structured CRM environment. The project shows the ability to manage stakeholders, define implementation scope, track adoption, reduce operational risk, and support a successful system rollout.

The CRM implementation improves customer data visibility, strengthens follow-up accountability, creates better reporting for managers, and gives leadership a clearer view of sales and service performance.
